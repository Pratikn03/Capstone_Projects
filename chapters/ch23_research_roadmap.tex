\chapter{ORIUS Program Architecture and Remaining Frontier}
\label{ch:roadmap}

This chapter treats ORIUS as one universal-first research program rather than
as a queue of disconnected papers. The purpose is to state the current
architecture of the contribution, identify which layers are already defended in
repo truth, and isolate the frontier that still blocks equal-domain
universality.

\section{The Universal Safety-Layer Spine}

The ORIUS spine now has one reader-facing structure:

\begin{enumerate}
  \item a universal degraded-observation hazard,
  \item a typed repair-and-certify runtime contract,
  \item a theorem layer that explains why the hazard matters and when repair is
    possible,
  \item six domain chapters that instantiate the same runtime object,
  \item and one governed parity gate that determines how strong the current
    cross-domain claim may be.
\end{enumerate}

That architecture is stronger than an older witness-centered roadmap because it
lets the reader understand ORIUS as a reusable safety layer from the start,
while still keeping the strongest proof language anchored to the deepest
available evidence row.

\section{Current Defended Layers}

The present package already contains several defended layers that belong in the
main thesis argument:

\begin{enumerate}
  \item \textbf{Reference witness depth.} The battery row remains the deepest
    theorem-to-code-to-artifact lineage and therefore carries the strongest
    proof-facing burden.
  \item \textbf{Temporal validity and bounded fallback.} Certificate horizon,
    expiration, blackout safe-hold, and graceful degradation are now part of
    the defended runtime story rather than future add-ons.
  \item \textbf{Benchmark discipline.} ORIUS-Bench supplies the replayable
    evaluation contract that keeps multi-domain comparison disciplined.
  \item \textbf{Bounded composition and bounded runtime governance.} Shared
    constraint coordination and the CertOS lifecycle now belong to the main
    scientific surface of the dissertation, albeit under bounded claim tiers.
\end{enumerate}

\section{What Remains Open}

The remaining work is no longer about inventing missing layers. It is about
closing explicit frontier rows:

\begin{enumerate}
  \item \textbf{Navigation.} The parity gate remains open until the real-data
    navigation row clears a full train/validate/replay chain with no synthetic
    fallback in the defended path.
  \item \textbf{Aerospace.} The row remains open until the placeholder surface
    is replaced by a real multi-flight safety task with non-degenerate replay
    and fallback evidence.
  \item \textbf{Equal-domain universality.} The book may only claim equal-peer
    universality after every row clears the same governed parity matrix.
  \item \textbf{Deployment beyond bounded runtime governance.} The current
    CertOS layer closes bounded lifecycle, audit, fallback, and recovery
    semantics; full field deployment remains outside the current evidence.
\end{enumerate}

\section{How to Read the Frontier Honestly}

The right scientific posture is therefore not to downplay ambition but to keep
it governed. ORIUS can now be presented as a universal-first monograph with a
strong common kernel and multiple defended instantiations, while the final
equal-domain claim remains tied to explicit closure obligations instead of
narrative pressure.

\section{Summary}

The ORIUS roadmap is now architectural rather than speculative. The universal
runtime layer, theorem ladder, benchmark contract, bounded composition, and
runtime governance surfaces already exist as defended parts of the dissertation.
The remaining frontier is concentrated in specific parity-gate obligations,
especially navigation and aerospace, not in a missing conceptual spine.

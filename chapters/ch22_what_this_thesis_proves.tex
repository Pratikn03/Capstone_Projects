\chapter{What This Thesis Proves and What It Leaves Open}
\label{ch:what-proves}

This chapter draws the boundary between the proved kernel of ORIUS and the
broader research program that surrounds it. The thesis is strongest when this
boundary is explicit.

\section{What Is Proved}

The current dissertation proves a domain-agnostic claim about degraded
observation:

\begin{enumerate}
  \item A controller can be observationally safe while physically unsafe when
    telemetry is stale, missing, delayed, or otherwise degraded.
  \item A repair-and-certify runtime layer can convert that hidden gap into a
    bounded control object through reliability-aware uncertainty inflation,
    safe-action repair, fallback, and certificate semantics.
  \item The full theorem-to-code-to-artifact depth of that claim is deepest on
    the battery witness row, where the proof ladder, replay evidence, and
    executable runtime traces align most completely.
\end{enumerate}

\section{What Is Defended Beyond the Witness Row}

The thesis also defends a broader universal program, but under bounded rather
than flattened claim tiers:

\begin{enumerate}
  \item industrial process control and healthcare monitoring are defended
    non-battery instantiations under the current replay and runtime contract;
  \item the bounded AV row contributes a defended autonomy instantiation under
    the current TTC entry-barrier contract;
  \item temporal validity, graceful fallback, benchmark discipline, bounded
    composition, and runtime governance are part of the defended ORIUS program
    rather than loose future ideas.
\end{enumerate}

\section{What Remains Outside the Current Evidence}

The thesis does \emph{not} yet prove:

\begin{enumerate}
  \item equal-domain closure across all six domain chapters,
  \item real-data navigation closure,
  \item a real multi-flight aerospace closure surface,
  \item heterogeneous cross-domain composition guarantees,
  \item or deployment-grade certification in live safety-critical hardware.
\end{enumerate}

\section{Why the Boundary Matters}

Being explicit about the boundary does not weaken the thesis. It protects the
main contribution. ORIUS is not interesting because it pretends every domain is
already equally mature. It is interesting because it identifies one hidden
safety hazard, one reusable runtime response, and one governed parity gate for
deciding how far each domain may currently be promoted.

\section{Summary}

The thesis proves a universal degraded-observation safety kernel with one
deepest witness row and several defended bounded extensions. It leaves the
remaining equal-domain frontier open in explicit, localized places rather than
blurring them into universal rhetoric.

\chapter{Five-Reviewer Committee Audit Protocol}
\label{app:four-reviewer-audit}

This appendix defines a structured audit protocol for the ORIUS/DC3S thesis.
Five independent reviewers assess distinct dimensions of the manuscript and
its supporting evidence.  The protocol now includes a dedicated reviewer for
the universal runtime and cross-domain harness so the promoted non-battery
claims are audited separately from the core battery evidence.

\section{Role Assignments}

\begin{description}
\item[R1 --- Empirical Correctness] Verifies that locked results match reported values, CSV integrity is preserved, and figure-to-data correspondence holds. Focus: \texttt{reports/publication/*.csv}, \texttt{dc3s\_main\_table\_ci.csv}, baseline lock manifests.
\item[R2 --- Theorem and Proof Rigor] Verifies theorem statements, proof steps, assumption usage, and mathematical notation consistency. Focus: ch15--ch19, appendices with full proofs.
\item[R3 --- Universal Runtime and Cross-Domain Harness] Verifies that the
  canonical universal replay harness, domain tiers, and promotion gate are
  internally consistent. Focus: \texttt{scripts/run\_universal\_orius\_validation.py},
  Chapter~\ref{ch:battery-to-orius}, and the generated domain status tables.
\item[R4 --- Reproducibility and Release Integrity] Verifies that scripts run,
  configs are complete, generated artifacts are current, and environment is
  documented. Focus: \texttt{scripts/}, \texttt{configs/},
  \texttt{requirements.txt}, release reports, and generated tables/figures.
\item[R5 --- Scope and Defense Coherence] Verifies claim scope accuracy,
  notation consistency, limitations, and committee-facing defense coherence.
  Focus: abstract, introduction, conclusion, scope boundary, and
  claim-to-evidence linkage.
\end{description}

\section{Procedure}

\begin{enumerate}
\item \textbf{Kickoff}: Author provides (a) thesis PDF, (b) \texttt{reports/publication/} tarball, (c) \texttt{scripts/} and \texttt{configs/} snapshot, (d) baseline lock manifest.
\item \textbf{Parallel review}: Each reviewer completes their assigned audit
  block using the generated checklist table and the linked evidence files.
\item \textbf{Reconciliation}: Failures are documented with specific line numbers, file paths, and suggested fixes. Deferred items require explicit justification.
\item \textbf{Re-audit}: After fixes, only failed items are re-checked. Full pass is required for submission.
\end{enumerate}

\section{Pass Criteria}

A manuscript passes the five-reviewer audit if:
\begin{itemize}
\item No R1 failures (empirical correctness is non-negotiable).
\item No R2 failures on the integrated 18-theorem release gate.
\item No R3 failures on the canonical universal replay harness or promoted
  proof-domain status table.
\item No R4 failures on artifact generation, release integrity, or script
  execution.
\item No R5 failures on scope boundary, limitations, or committee-facing
  defense coherence.
\end{itemize}

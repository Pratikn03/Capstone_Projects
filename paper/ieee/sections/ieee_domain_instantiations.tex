\section{Domain Instantiations under One Runtime Contract}
\label{sec:ieee-domains}

The key test of a universal safety layer is not whether every domain shares the same
dynamics or controller. It is whether the same runtime contract survives domain change
without losing semantic meaning. ORIUS evaluates that question across the defended
Battery, AV, and Healthcare rows.

\subsection{Autonomous vehicles}
The AV row is defended under a bounded safety object: time-to-collision and predictive
entry-barrier preservation when the perception or state-estimation surface degrades. The
current evidence lane uses completed all-zip grouped nuPlan replay artifacts, not a full road-deployment
claim. The value of the row is that ORIUS is not merely filtering a scalar dispatch
command. It repairs an action inside a tightly constrained interaction envelope while
preserving bounded fallback semantics. The row is defended, but only for the current
TTC plus predictive-entry-barrier contract rather than advertised as full
autonomous-driving field closure.


\subsection{Healthcare monitoring}
Healthcare matters because degraded observation can suppress intervention rather than
only create excessive actuation. ORIUS still fits that setting because the runtime layer
is defined over admissible decisions, fallback policies, and certificate semantics, not
only over physical force commands. The defended healthcare row is bounded to
retrospective MIMIC-backed monitoring and alerting contracts. It does not claim
prospective clinical deployment or clinical decision-support approval, but it shows
that the runtime grammar is not limited to mechanical control loops.

\subsection{Future additional domains}
The active branch does not defend named additional domains beyond Battery, AV, and
Healthcare. Future extensions remain part of the architectural argument, but they are
not part of the current empirical claim surface until they clear the same replay,
runtime, governance, and bounded-prose obligations as the active three-domain lane.

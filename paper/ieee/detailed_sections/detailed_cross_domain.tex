\subsection{Analysis}

\paragraph{TSVR.}  The battery domain achieves the ideal $\TSVR = 0$ because the
safety constraints (SoC bounds) are hard and the controller operates well within
margins.  The AV domain has a higher TSVR (0.628) because the prediction task is
fundamentally harder: ego speed at 4\,s horizon involves complex multi-agent
interactions that the LightGBM model cannot perfectly forecast.  Importantly,
ORIUS \emph{reduces} the AV TSVR from 0.660 (baseline) by 4.8\%.

\paragraph{Intervention rate.}  The dramatic difference---0--2\% (battery) \vs\ 90\%
(AV)---reflects the gap between domains.  In the battery domain, the baseline
controller is already safe, so the shield rarely intervenes.  In the AV domain,
degraded perception routinely pushes the candidate action outside the tightened
constraint set, requiring frequent repair.

\paragraph{CVA uniformity.}  Both domains achieve CVA $\ge 0.979$, indicating that
the conformal calibration mechanism generalises across domains without per-domain
tuning.

\paragraph{Governance uniformity.}  Both domains achieve 100\% chain validity and
audit completeness, demonstrating that the governance surface is domain-agnostic.

\subsection{Kernel Invariance}

A critical design goal of ORIUS is that the kernel binary is unchanged across
domains.  To verify this, we confirm:
\begin{enumerate}[nosep]
  \item The same \texttt{kernel.py} file is used for both domains (SHA-256 match).
  \item The same \texttt{contracts.py} file is used for both domains.
  \item The same \texttt{ContractVerifier} predicates (CP1--CP6) are evaluated.
  \item Only the adapter modules (\texttt{battery\_adapter.py} \vs\
        \texttt{av\_adapter.py}) differ.
\end{enumerate}

This validates the architectural claim: domain-specific behaviour is isolated in the
adapter; the kernel and contract stack are universal.

\subsection{Coverage Improvement Pattern}

Across both domains, the conformal widening mechanism lifts coverage from the base
level (0.87--0.94) to the widened level ($\ge 0.95$).  The inflation cost
(interval width increase) is approximately:
\begin{itemize}[nosep]
  \item Battery: 375--1{,}107~MWh (proportional to fault severity).
  \item AV: 3--264~m/s or m (proportional to fault severity and target).
\end{itemize}

The inflation factor is governed by the same formula (\cref{eq:inflation}) in both
domains, with the same parameters ($0.7$ for degradation weight, $0.5$ for shift
weight, clipped to $[1, 3]$).

\subsection{Implications for Universality}

The two-domain evaluation provides strong evidence that ORIUS's universal design is
not merely aspirational but practically realisable.  The same typed contracts, the
same five-stage pipeline, and the same governance surface successfully handle:
\begin{itemize}[nosep]
  \item A \textbf{slow-timescale energy system} (15-minute intervals, 168-hour
        episodes) with scalar state.
  \item A \textbf{fast-timescale vehicle system} (0.1\,s intervals, $\sim$4\,s
        episodes) with multi-dimensional state.
\end{itemize}

The adaptation cost is confined to implementing six adapter methods per domain.

bound is not tight due to multi-agent prediction difficulty).

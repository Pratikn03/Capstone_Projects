\section{Autonomous Vehicle Validation}\label{sec:av}

This section provides a comprehensive evaluation of ORIUS on the Autonomous Vehicle
(AV) longitudinal prediction and safety domain.  The AV adapter wraps a LightGBM
conformal predictor trained on the Waymo Open Motion Dataset.

\subsection{Domain Description}

The AV domain addresses \emph{longitudinal safety prediction}: given the ego
vehicle's telemetry and surrounding traffic context, predict the ego speed and
relative gap to the lead vehicle at horizons of 1\,s, 2\,s, and 4\,s.  The
observation may be degraded by perception pipeline faults (latency, dropout, drift,
reordering, spikes, staleness).

The safety constraints for this domain are:
\begin{enumerate}[nosep]
  \item Minimum following gap: $\mathrm{gap}_t \ge \mathrm{gap}_{\min}$.
  \item Speed bounds: $0 \le v_t \le v_{\max}$.
  \item Conformal coverage: the true future state must be contained in the
        uncertainty set with probability $\ge 1 - \alpha_{\mathrm{target}}$.
\end{enumerate}

\subsection{LightGBM Model Configuration}

The quantile regression model is a gradient-boosted decision tree ensemble:

\begin{itemize}[nosep]
  \item \textbf{Library}: LightGBM 4.x with quantile objective.
  \item \textbf{Quantile levels}: $\tau = 0.025$ (lower), $\tau = 0.975$ (upper).
  \item \textbf{Number of trees}: 500 (early stopping patience 20).
  \item \textbf{Max depth}: 6.
  \item \textbf{Learning rate}: 0.05.
  \item \textbf{Feature fraction}: 0.8 per split.
  \item \textbf{Min data in leaf}: 20.
\end{itemize}

Separate models are trained for each (target, horizon) pair, yielding 12 models
total (2~targets $\times$ 3~horizons $\times$ 2~quantile levels).

\subsection{Conformal Calibration Procedure}

Split conformal prediction is applied on the calibration split (205~scenarios):

\begin{enumerate}[nosep]
  \item Compute non-conformity scores: $s_i = \max(y_i - \hat{q}_{0.975}(x_i),\;
        \hat{q}_{0.025}(x_i) - y_i)$ for each calibration point.
  \item Sort scores and take the $\lceil (1 - \alpha)(n + 1) \rceil$-th smallest as
        the conformal radius $\hat{r}$.
  \item The base conformal interval is $\Chat(x) = [\hat{q}_{0.025}(x) - \hat{r},\;
        \hat{q}_{0.975}(x) + \hat{r}]$.
  \item Apply adaptive widening: $\Crac(x) = \alpha \cdot \Chat(x)$ with $\alpha$
        computed from \cref{eq:inflation}.
\end{enumerate}

The widening factor of $\approx 1.22$ lifts all coverage values above 0.95.

\subsection{Per-Fault TSVR Analysis}

While the aggregate ORIUS TSVR is 0.628, its distribution varies across fault families.
Fault families that cause the largest observation--reality divergence (\eg~spikes, drift)
trigger stronger inflation and more frequent shield interventions, keeping the TSVR bounded.
Conversely, mild faults (delay jitter, nominal reordering) require minimal intervention
and show TSVR close to the baseline.

The key insight is that the inflation formula (\cref{eq:inflation}) provides
\emph{automatic scaling}: severe faults lower $\wt$ significantly, which increases $\alpha$,
which widens $\Crac$, which tightens $\Aset$.  This cascading effect is the mechanism by
which ORIUS achieves degradation-proportional conservatism (Theorem T8).

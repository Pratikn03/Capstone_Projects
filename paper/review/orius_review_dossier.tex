\chapter{Purpose and Protocol}
This dossier records the canonical simulated 15-reviewer audit for the live
ORIUS defense package. The package is grounded in repo truth: the active
three-domain lane (Battery + AV + Healthcare), the machine-generated active
theorem audit, the canonical build path \texttt{make review-compile}, and the
generated publication artifacts under \texttt{reports/publication/}.

ORIUS is defended as one bounded stack:
\begin{itemize}
\item \textbf{Phenomenon}: the Observation--Action Safety Gap (OASG),
\item \textbf{Mechanism}: reliability-aware runtime safety and calibration under degraded observation,
\item \textbf{Evidence lane}: Battery as witness row, AV and Healthcare as bounded defended peer rows.
\end{itemize}

The single flagship novelty sentence used across the defense package is:
\begin{quote}
\emph{ORIUS identifies OASG as the degraded-observation release hazard and provides a reliability-aware runtime safety layer across Battery, AV, and Healthcare.}
\end{quote}

\section*{Novelty boundary}
The novelty claim is architectural and runtime-semantic: ORIUS does not claim a new
conformal method, a new robust-optimization primitive, a new universal controller, or a
new conditional-coverage theorem.

Reviewer-facing separation is explicit:
\begin{itemize}
\item conformal methods contribute uncertainty sets, but not repaired actuation or release certification;
\item runtime monitors and anomaly detectors contribute alarms or vetoes, but not tightened governed release;
\item simplex and runtime-assurance schemes contribute supervisory override and fallback, but not reliability-conditioned tightening under one shared release contract; and
\item safety filters and robust MPC contribute plant-local safe action geometry, but not a reusable degraded-observation release grammar across domains.
\end{itemize}

The flagship ML surface is grouped calibration plus runtime safety under degraded observation. This dossier does not defend a new conditional-coverage theorem and does not take headline novelty credit from draft theorem rows.

Every review finding must end in exactly one disposition:
\begin{itemize}
\item \textbf{fixed in code/tests},
\item \textbf{narrowed in prose/registers}, or
\item \textbf{left open as bounded future work}.
\end{itemize}

\chapter{Final Disposition Summary}
The canonical defense package does \emph{not} claim that every theorem-like row
in the repository is equally rigorous. It claims something narrower and more
defensible:
\begin{enumerate}
\item the live defended program is the promoted Battery + AV + Healthcare lane,
\item the registry-canonical theorem core is explicitly classified into flagship, supporting, and draft surfaces,
\item proof rigor is tracked by the active theorem audit rather than by a traceability gate pretending to be a proof checker, and
\item any row that still lacks aligned code, tests, proofs, and bounded scope wording is demoted rather than rhetorically preserved.
\end{enumerate}

That is the repo-truthful ORIUS defense posture after hardening: a smaller,
auditable, reviewer-safe claim surface intended to survive hostile reading.
